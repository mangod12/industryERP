# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| 1.x     | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within Kumar Brothers Steel ERP, please report it responsibly:

1. **Do not disclose publicly** until the issue is resolved
2. **Email**: security@kumarbrothers.com
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will acknowledge receipt within 48 hours and provide a timeline for resolution.

## Security Best Practices

1. **Change default credentials** immediately after installation
2. **Use HTTPS** in production
3. **Keep dependencies updated**: `composer update`
4. **Enable 2FA** for admin accounts
5. **Regular backups** of database and uploads
6. **Review access logs** periodically
